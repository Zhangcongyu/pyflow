
重新进行梳理：
Phase 0: 需求互动
这个环节无论简单或者复杂都需要，通过这个阶段，明确是否新建，以及用户需求

Phase 1: Constitution（项目规则）
如果不是新建，是旧项目新增功能，此阶段可以跳过

Phase 2: Specify + Clarify（功能规格）
应该输入第0阶段生成的用户需求

Phase 3: Plan（技术方案）
 
Phase 4: Tasks（任务分解）
这个Tasks包括了目录创建、依赖管理、日志管理、git管理、项目编码、代码审核、最终集成测试等全套步骤
详见spec-kit-example这个文件夹下的例子，这时以前使用spec-kit产生的内容，如tasks.md
所以从以下5-9的阶段都不存在
应该新建一个执行的命令，根据Tasks,逐项完成，特别是编码、测试等步骤，应该引入TDD

或者，取消这个阶段，建立自己的任务
 
Phase 5: TDD 实现
Phase 6: (代码审查)
Phase 7: (质量保证)
Phase 8: (文档)
Phase 9: Git 分支清理
